# OctoAcme Project Management Docs

Welcome to the OctoAcme project management documentation hub. This folder contains all process guides that define how OctoAcme plans, executes, and delivers projects. Whether you are a new team member getting oriented or a practitioner looking for a specific process reference, this README is your starting point.

## Summary of Project Management Processes at OctoAcme

OctoAcme follows an iterative, customer-first delivery methodology built on four key principles:

- **Iterative delivery** – work is broken into small, testable increments to reduce risk and accelerate feedback.
- **Customer-first value** – every prioritization decision starts with the impact on customers and end users.
- **Clear ownership** – each project has a named Project Manager (PM) and Product Manager (PdM) who are accountable for delivery and outcomes respectively.
- **Data-informed decisions** – success metrics are defined upfront and used to evaluate impact and guide continuous improvement.

Projects move through a clearly defined lifecycle:

**Initiation → Planning → Execution → Release → Retrospective**

Throughout this lifecycle, OctoAcme emphasizes open communication (weekly syncs, standups, and stakeholder updates), measurable outcomes tied to customer and business goals, and a commitment to continuous improvement through structured retrospectives.

Roles are clearly defined across four primary personas — **Project Manager (PM)**, **Product Manager (PdM)**, **Developers**, and **QA/Testing** — each with explicit responsibilities, goals, and communication norms documented in the [Roles and Personas](octoacme-roles-and-personas.md) guide.
